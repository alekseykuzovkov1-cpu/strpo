### 2. Настройка удаленного доступа

* В качестве клиента для подключения по протоколу **SSH** был выбран **Git Bash**, который уже был ранее установлен

* Был настроена служба **OpenSSH** следующим образом:
    * Установка пакета:
        ```
        sudo pacman -Syu openssh
        ```
    * Запуск службы и добавление её в автозагрузку:
        ```
        sudo systemctl enable --now sshd
        ```

* Порт TCP — логический номер, используемый в сетевых протоколах для идентификации процесса или приложения на хосте. Позволяет операционной системе распределять входящие сетевые пакеты между запущенными программами

    [Источник](https://ru.wikipedia.org/wiki/TCP)

* Проброс портов - механизм перенаправления входящих сетевых запросов с определенного порта хостовой машины (например, 127.0.0.1:2222) на порт конкретного сетевого узла или виртуальной машины во внутренней сети (например, 10.0.2.15:22). Это необходимо для организации доступа к сервисам (в данном случае SSH) внутри изолированной среды виртуализации при использовании сетевого режима NAT

    [Источник](https://realinternet.ru/blog/probros-portov/)

* Было настроено пробрасывание портов следующим путем:
    * У виртуальной машины нажать "Настроить" -> "Расширенные" -> "Сеть" -> "Проброс портов"
    * Добавить новое правило:
        * Имя: SSH

        * Протокол: TCP

        * Адрес хоста: 127.0.0.1

        * Порт хоста: 2222

        * Адрес гостя: оставить пустым

        * Порт гостя: 22

* Подключение по паролю было осуществленно в **Git Bash** с помощью команды:
    ```
    ssh aleksey1@127.0.0.1 -p 2222
    ```

* Добавление публичного ключа на виртуальную машину было сделано в два этапа:
    1. Генерация пары ключей на Windows с помощью команды `ssh-keygen -t rsa -b 4096` в **Git Bash**
    2. Копирование публичного ключа на виртуальную машину командой `ssh-copy-id -p 2222 aleksey1@127.0.0.1` в **Git Bash**
    * Теперь при подключении к серверу через `ssh aleksey1@127.0.0.1 -p 2222` сервер пускает без запроса пароля

        [Источник](https://habr.com/ru/articles/938968/)

* Был отключен доступ по паролю следующим образом:
    * После подключения к виртуальной машине по SSH используем команду `sudo nano /etc/ssh/sshd_config` и открываем файл конфигурации **sshd_config**, в нем находим строку **PasswordAuthentication** и изменяем ее на **PasswordAuthentication no**, сохраняем изменения
    * Проверить отключение доступа можно командой `ssh aleksey1@127.0.0.1 -p 2222 -o PreferredAuthentications=password -o PubkeyAuthentication=no`. При выполнении команды сервер выдает ошибку **Permission denied (publickey)**, а это означает, что возможность входа для всех, у кого нет ключа, успешно закрыта
