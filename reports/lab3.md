#### 2.5. Настройка приглашения ко вводу в командной строке, примеры информации в приглашении ко вводу
+ Механизм настройки

    В PowerShell за внешний вид строки ввода отвечает функция с именем prompt. По умолчанию она просто возвращает путь к текущей директории и символ >. Чтобы изменения сохранялись после перезагрузки консоли, функцию нужно прописать в профиле пользователя:
    1. в PowerShell: notepad $PROFILE
    2. Вставить в открывшийся файл код функции prompt
    3. Сохранить и перезапустить консоль

+ Примеры выводимой информации:
    + **Текущее время**: помогает отслеживать, когда была выполнена команда

    + **Ветка Git**: позволяет не вводить постоянно git branch

    + **Статус администратора**: можно менять цвет текста, если консоль запущена от имени администратора

    + **Результат последней команды**: если программа упала, приглашение может окраситься в красный цвет

    + **Сокращенный путь**: вместо длинной строки C:\Users\aleks\Desktop\strpo\reports показывать только strpo\reports

#### 2.6. Настройка конфигурации
```
function prompt {
    # получаем текущее время
    $time = Get-Date -Format "HH:mm:ss"
    
    # сокращаем путь (оставляем только последние две папки)
    $currentPath = (Get-Location).Path
    $shortPath = Split-Path $currentPath -Leaf
    $parentPath = Split-Path (Split-Path $currentPath -Parent) -Leaf
    $displayPath = if ($parentPath) { "$parentPath\$shortPath" } else { $shortPath }

    # получаем ветку Git
    $gitBranch = git branch --show-current 2>$null
    $gitStatus = if ($gitBranch) { " [$gitBranch]" } else { "" }

    # проверяем, была ли ошибка в прошлой команде
    $lastErrorColor = if ($global:?) { "Green" } else { "Red" }

    # формируем вывод
    Write-Host "[$time] " -ForegroundColor Gray -NoNewline
    Write-Host "$displayPath" -ForegroundColor Cyan -NoNewline
    Write-Host "$gitStatus" -ForegroundColor Yellow -NoNewline
    Write-Host " >" -ForegroundColor $lastErrorColor -NoNewline
    
    return " "
}
```

### 3. Эргономика работы с кодом
#### 3.1. Cписок горячих клавиш для редактирования текста в VS Code
* Манипуляции со строками
    * Ctrl + Enter - вставить строку ниже и перейти на неё
    * Shift + Enter - вставить строку выше
    * Alt + ↑ / ↓ - переместить строку вверх или вниз
    * Shift + Alt + ↓ / ↑ - дублировать строку

* Навигация и редактирование
    * Ctrl + D - выделить следующее вхождение слова
    * Ctrl + Shift + \ - перейти к парной скобке
    * Ctrl + / - закомментировать/раскомментировать строку

* Работа со структурой проекта
    * Ctrl + P - быстрый переход к файлу
    * Ctrl + Shift + O - переход к символу (методу или переменной)

#### 3.2. Выполнение конкретных действий
* поиск по содержанию файлов: Ctrl + Shift + F
* перейти к объявлению функции или типа: F12 или Ctrl + ЛКМ
* перейти к реализации функции или типа: Ctrl + F12
* найти все места, где используется переменная, функция или тип: Shift + F12
* переименовать все вхождения переменной, функции или типа: F2
* быстро перемещаться между несколькими наиболее интересными местами кода:
    * История переходов: Alt + <- и Alt + ->
    * Закладки:
        * Ctrl + Alt + K — поставить/снять закладку
        * Ctrl + Alt + L — быстро прыгнуть к следующей закладке
* комментировать и раскомментировать большие части кода: 
    * Строчный комментарий (//): выделить блок и нажать 
    Ctrl + /

    * Блочный комментарий (/* ... */): выделить блок и нажать Shift + Alt + A

#### 4.2. Биндим кнопки
1. Собирать программу в нужной среде (например, в WSL)

* Чтобы собирать программу внутри WSL, не выходя из VS Code, нужно создать задачу, которая делегирует выполнение команд оболочке WSL

* Настройка tasks.json: в Terminal -> Configure Tasks добавлено:
```
{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "Build in WSL",
            "type": "shell",
            "command": "wsl",
            "args": ["g++", "-g", "${file}", "-o", "${fileBasenameNoExtension}"],
            "group": {
                "kind": "build",
                "isDefault": true
            }
        }
    ]
}
```
* Сочетание клавиш: по умолчанию Ctrl + Shift + B запустит задачу сборки.

* Кнопка в интерфейсе: было установлено расширение "Task Buttons". В settings.json добавлено:
```
"VsCodeTaskButtons.tasks": [
    { "label": "Build WSL", "task": "Build in WSL" }
]
```
После этого кнопка появилась в строке состояния внизу

2. Запускать программу в режиме отладки или релиза
* Для этого нужно изменить файл launch.json. Режим можно будет выбрать в выпадающем списке на панели отладки

* Настройка launch.json: 
```
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "debug",
            "type": "cppdbg",
            "request": "launch",
            "program": "${workspaceFolder}/${fileBasenameNoExtension}",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${workspaceFolder}",
            "environment": [],
            "externalConsole": false,
            "MIMode": "gdb",
            "preLaunchTask": "Build in WSL" // сначала соберет, потом запустит
        },
        {
            "name": "release",
            "type": "cppdbg",
            "request": "launch",
            "program": "${workspaceFolder}/${fileBasenameNoExtension}",
            "args": ["--release-mode"], 
            "MIMode": "gdb"
        }
    ]
}
```
* Сочетание клавиш: F5 - запуск текущего выбранного режима, Ctrl + F5 - запуск без отладчика

### 5. Интеграция с гитом
#### 5.1. Настройка интеграции с гитом в VS Code
VS Code автоматически подхватывает установленный в системе Git. Для авторизации в "Учетной записи" было выбрано "Sign in to sync settings", что позволило делать push и pull без постоянного воода пароля. Во вкладке Source Control (Ctrl + Shift + G) виден список измененных файлов, что говорит о том, что Git установлен
#### 5.2. Команды Git в VS Code
1. Просмотр изменений (git diff)
* Клик по названию файла в списке Changes на панели Source Control

2. Добавление файлов в индекс (git add)
* Навести курсор на файл или папку в списке Changes и нажмите на иконку "+" (Stage Changes)

3. Коммиты (git commit)
* Ввести описание изменений в текстовое поле сверху (над кнопкой Commit), затем нажать кнопку Commit

4. Добавление удаленного репозитория (git remote add)
* Нажать на три точки (...) в верхней части панели Source Control, затем:
    1. Выбрать пункт Remote -> Add Remote....
    2. Вставить URL репозитория и дать ему имя

5. Получение обновлений (git pull/fetch)
* Нажать на иконку "зацикленных стрелок" в строке состояния внизу слева или выбрать Pull в меню трех точек

6. Отправка обновлений (git push)
* После коммита появится кнопка Sync Changes, нужно нажать на неё

7. Разрешение конфликтов (Merge Conflicts)
* Если при получении обновлений возник конфликт, файлы в списке подсветятся буквой C (Conflict). При открытии таких файлов прямо над конфликтующим кодом появятся кнопки:

    * Accept Current Change (Оставить вариант)

    * Accept Incoming Change (Взять вариант из облака)

    * Accept Both Changes (Сохранить оба варианта)

* Merge Editor: в правом нижнем углу находится кнопка "Resolve in Merge Editor", которая откроет удобное трехпанельное окно для сложного разбора конфликтов

[Источник](https://code.visualstudio.com/docs/sourcecontrol/overview)
